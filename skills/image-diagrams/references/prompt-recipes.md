# Imagegen 图解提示词配方

先把用户材料整理成内容契约，再选择一个配方。只填来源已经给出的内容；不为画面完整而补写研究发现、数字、关系或标签。

## 通用内容契约

```text
用途与载体：论文 / 报告 / PPT；全页 / 半页 / 单栏 / 双栏
图类：流程 / 技术路线 / 框架 / 机制 / 结构 / 包含 / 时间线 / 架构 / 场景配图
画布：横向或纵向；目标宽高比
阅读顺序：从左到右 / 从上到下 / 中心向外
节点：ID | 逐字标签 | 层级 | 视觉对象
关系：起点 ID -> 终点 ID | 方向 | 关系类型
必留：必须出现的对象、颜色或留白区
禁加：来源没有的文字、数字、因果箭头、图标、标题和装饰
```

## 流程图与技术路线

```text
Use case: infographic-diagram
Asset type: <论文/报告/PPT> 流程图
Primary request: 根据下列节点和关系生成专业的 <横向/纵向> 流程图，阅读顺序为 <方向>。
Composition/framing: <宽高比>；主路径清楚；分支靠近来源节点；留白充分。
Text (verbatim): 只使用以下逐字标签：<标签列表>。
Constraints: 节点与边必须与下表一一对应；箭头方向准确；不得新增或合并步骤；背景干净；无水印。
Avoid: 额外标题、解释句、伪文字、重复节点、交叉箭头、卡片墙、随机图标、装饰性渐变。
```

对样本纳排、队列筛选、CONSORT 或病例流转，把每个样本量和排除原因列为逐字标签。签发前逐项比对全部数字和分支；任一项不准即用 imagegen 定向修正或整图重生成，多轮后仍不准确才整图回退 SVG。

## 概念框架与机制示意

```text
Use case: scientific-educational
Asset type: <论文/报告/PPT> 概念框架或机制示意
Primary request: 用克制、清晰的科学插图表达 <核心命题>，严格依据节点与关系表组织视觉层级。
Subject: <真实存在的对象、组织、器官、暴露、过程或结局>。
Style/medium: 高质量科学教育插图；专业、准确、非营销化；背景简洁。
Composition/framing: <左-中-右/中心汇聚/分层>；<宽高比>；为标签保留明确空间。
Text (verbatim): 只使用 <标签列表>，逐字准确呈现全部标签；不得省略、改写或生成伪文字。
Constraints: 不改变方向和层级；不添加未提供的机制、通路、数据或结论；无水印。
Avoid: 伪文字、夸张因果、卡通化器官、无依据分子结构、装饰图标、霓虹渐变。
```

## PPT 研究配图

```text
Use case: scientific-educational
Asset type: 学术 PPT <背景/方法/机制/场景> 配图
Primary request: 生成可直接用于学术汇报的 <主题> 配图，视觉重点为 <主体>。
Style/medium: 专业科学插图或写实科研场景，与现有模板色彩协调。
Composition/framing: <16:9/实际比例>；主体位于 <位置>；为页面正文在 <位置> 留出干净负空间。
Text (verbatim): 无图内标题、无解释性段落；只有用户明确要求时才加入逐字标签。
Constraints: 主体完整、远离裁切边缘；不出现品牌标识、界面伪文字或水印。
Avoid: 宣传海报感、库存照片感、过暗背景、过度景深、装饰光斑、无关实验器材。
```

## 定向修正

每轮继续通过 imagegen 修正完整成图，只修一个问题并重复不变量：

```text
只修正：<单一问题>。
保持不变：构图、比例、主体、节点数量、关系、配色、其它逐字标签。
不得新增、删除、移动或改写其它内容。
```

修正后重新检查整张原图。不得用 Python 后处理、PPT/Word 文本框或 SVG 覆盖层补写文字、数字和箭头；多轮后仍不准确时，整图回退 `svg-diagrams`。
