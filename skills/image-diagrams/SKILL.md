# Imagegen 图解与配图

以可用的文生图工具作为非统计图解与 PPT 配图的默认生成路径。Codex 使用内置 `imagegen` 技能及其 `image_gen` 工具；不要另写 API 脚本，也不要猜模型、参数或命令。其它平台只使用当前会话明确提供且已配置的图像生成工具；没有可用工具时直接调用 `svg-diagrams`。

## 必读资源

- 每次生成前阅读 `references/prompt-recipes.md`，按图类选择提示词骨架。
- Codex 同时遵循内置 `imagegen` 的生成、编辑、保存路径和透明背景规则。
- 统计图、森林图、生存曲线、热图和其它数据映射图不进入本技能，调用 `publication-figures`。

## 路由门禁

按以下顺序判定，不因流程图或框架图看似可用简单形状表达就跳过 imagegen：

1. 用户明确要求 SVG、矢量、完全可编辑文字或期刊指定矢量源文件时，调用 `svg-diagrams`。
2. 当前会话有可用文生图工具时，默认走 imagegen，包括流程图、框架图、机制图、示意图和 PPT 配图。
3. 文生图工具不可用、调用失败且没有可用的内置路径，或生成后仍不能满足内容精度门禁时，调用 `svg-diagrams`。
4. 文字或数字较多不构成预先跳过 imagegen 的理由。先生成并核验；若画面质量合格但文字不稳，优先保留 imagegen 视觉层并叠加确定性文字，再考虑整图 SVG 回退。

不得把 CLI/API 当作内置工具失败后的静默替代。是否允许 CLI、是否需要凭证及可选模型均服从 `imagegen` 自身规则和用户明确指示。

## 工作流

1. **锁定内容契约**：列出用途、载体、比例、语言、图类、节点、逐字标签、关系、方向、必留元素和禁加元素。来源没有的标题、数字、因果关系、图标或解释不得补写。
2. **选择生成方式**：
   - 流程、框架、机制、结构和科学示意采用 `infographic-diagram` 或 `scientific-educational`。
   - PPT 背景、研究场景和概念配图采用最匹配的 imagegen use case；默认不在图内写页标题或解释性段落。
   - 需要精确文字的图可先生成完整图；若首轮文字误差明显，下一轮改为无文字视觉层或留出标签区，再以 PPT/Word/SVG 的可编辑文字覆盖。
3. **构造提示词**：把内容契约写成 `Primary request / Composition / Text (verbatim) / Constraints / Avoid`。明确画布比例、阅读顺序、节点和边关系，要求不增加内容、无水印、无装饰性乱码。
4. **生成初稿**：Codex 直接调用内置 `image_gen`。项目产物生成后移入项目工作区；不得只留在 `$CODEX_HOME/generated_images/`。
5. **逐项核验**：用图像查看工具按原始分辨率检查，再按最终嵌入尺寸检查。逐字比对标签、数字和缩写；逐条比对节点、箭头、方向和层级。不能只凭整体观感签发。
6. **定向修正**：每轮只修一个明确问题并重新核验。若完整图的文字仍不可靠，转为 imagegen 视觉层加确定性文字覆盖；若关系、方向或关键元素仍错误，整图转 `svg-diagrams`。
7. **嵌入载体**：PPT 和 Word 默认嵌入高质量 PNG；保持原始比例，不拉伸。论文按目标版面和期刊要求导出，若期刊强制矢量则走 SVG 回退。
8. **记录与归档**：正式项目记录最终提示词、采用的工具路径、核验结论和回退原因；当前版使用稳定文件名，旧版整组归档。不得把提示词或生成过程写入论文、报告或 PPT 正文。

## 质量策略

### 完整 imagegen 图

适用于标签少、结构清楚、视觉表达占主导的流程图、机制图、概念框架和科学示意。要求：

- 白色或与载体一致的干净背景，清楚的视觉层级和充足留白。
- 只出现内容契约中的文字；不自动添加英文副标题、图例、徽章或解释句。
- 连接关系清晰，箭头不穿过文字或主体，不增加伪因果关系。
- 不使用大面积装饰渐变、彩色卡片墙、随机 3D 图标或与主题无关的场景元素。

### Imagegen 视觉层加确定性文字

适用于画面风格和科学对象由 imagegen 提升明显，但标签、数字、公式或路径必须逐字准确的图：

- 提示 imagegen 生成无文字版本、清晰留白区或可覆盖的标签位。
- 在 PPT 中优先用原生文本框和连接线叠加；论文或报告可用 SVG 透明覆盖层后再导出 PNG。
- 覆盖层只承担文字、数字、公式和必要连接，不重画整个视觉主体。
- 最终按合成后的成品核验，不把未合成底图当成可交付图。

### SVG 整图回退

仅在以下情形使用：

- 当前会话没有可用文生图工具。
- 用户或目标格式明确要求 SVG、矢量源或完全可编辑图形。
- imagegen 调用失败，且其自身允许的重试或路径仍不可用。
- 经定向修正或混合覆盖后，标签、数字、关系、方向或关键对象仍不准确。

回退时把已锁定的内容契约原样交给 `svg-diagrams`，不得在切换工具时改动信息结构。

## 载体要求

- **PPT 全页图**：按实际占位生成 16:9 或相应宽高比；页标题由模板承担，图内不重复标题。左右图文页为正文留出可用负空间。
- **论文图**：按单栏或双栏最终尺寸检查文字可读性；无期刊依据时不声称生成图是矢量图。
- **报告图**：按 Word 版心宽度生成，使用 PNG 嵌入；图题留在正文，不烧录进图片。
- **透明素材**：严格按 `imagegen` 的透明背景流程执行，不自行假设工具支持原生透明。

## 强制自检

- [ ] 已确认 imagegen 是否可用；可用时未因图类简单而直接跳到 SVG
- [ ] 文字、数字、缩写、节点、关系和方向与来源逐项一致
- [ ] 无虚构内容、乱码、伪文字、水印、重复标题或无语义装饰
- [ ] 画布比例匹配最终载体，主体完整且没有被裁切或拉伸
- [ ] 在原始分辨率与最终嵌入尺寸下均已目视检查
- [ ] 文字不稳时已采用确定性覆盖或 SVG 回退，而非带错签发
- [ ] 项目引用的最终图片已进入工作区并使用稳定文件名
- [ ] 正式项目已记录最终提示词、工具路径、核验结论和回退原因
