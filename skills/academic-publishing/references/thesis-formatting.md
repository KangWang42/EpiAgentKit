## 1 最高法与默认值边界

**本校研究生院《学位论文格式规范》覆盖本文件一切默认值**；用户给了字体/字号/页边距/行距/参考文献格式的具体
要求时同样优先。下表是 GB/T 7713.1 + 国内高校通行基线，**未拿到学校规范时用作默认，并在交付时标注"排版默认
值，待以学校规范核对"**。字体/字号/格式应做成可按用户要求替换的参数，不写死。

## 2 页面设置

| 项 | 默认值（基线） |
|---|---|
| 纸张 | A4（210×297 mm），单面或双面按学校 |
| 页边距 | 上 2.5 / 下 2.5 / 左 3.0 / 右 2.0 cm（装订侧留宽；以学校为准） |
| 行距 | 正文 1.5 倍行距（或固定值 20–22 pt） |
| 页眉 | 学校名 / 论文题名，五号宋体居中，下加细线 |
| 页码 | **前置部分（摘要/目录）用罗马数字 Ⅰ Ⅱ Ⅲ；正文起用阿拉伯数字 1 重新计数**；页脚居中 |
| 分页 | 封面 / 声明 / 摘要 / 目录 / 每一章 / 参考文献 / 综述 / 致谢 / 附录 各自**另起页**（分页符） |

## 3 字体字号表（逐部件，基线）

| 部位 | 中文字体 | 西文/数字 | 字号 | 其他 |
|---|---|---|---|---|
| 封面题名 | 黑体 | Times New Roman | 小一/二号 | 居中、加粗 |
| 章标题（第 N 章） | 黑体 | Times New Roman | 三号 | 居中、段前后空行、另起页 |
| 一级节 1.1 | 黑体 | Times New Roman | 四号 | 左对齐、加粗 |
| 二级节 1.1.1 | 黑体 | Times New Roman | 小四 | 左对齐、加粗 |
| 正文 | 宋体 | Times New Roman | 小四（12pt） | 首行缩进 2 字符、1.5 倍行距、两端对齐 |
| 摘要"摘 要"标题 | 黑体 | — | 三号 | 居中 |
| 图题 / 表题 | 黑体或宋体加粗 | Times New Roman | 五号（10.5pt） | 图题在图下居中、表题在表上居中 |
| 表内文字 | 宋体 | Times New Roman | 小五（9pt） | — |
| 表注 / 图注 | 宋体 | Times New Roman | 小五 | 在图表下方 |
| 参考文献 | 宋体 | Times New Roman | 五号 | 悬挂缩进 |
| 页眉 | 宋体 | Times New Roman | 五号 | 居中 |

实现关键（双字体，见 `docx-assembly.md` §2）：
```python
run.font.name = 'Times New Roman'
run._element.rPr.rFonts.set(qn('w:eastAsia'), '宋体')   # 中文走 eastAsia
```
所有字号/字体抽成脚本顶部常量字典，便于按学校规范/用户要求一处改全局。

## 4 章节层级与图表公式编号

- **章节**：`第 1 章 前言` → `1.1` → `1.1.1`（三级为主，必要时 1.1.1.1）。章用"第 N 章"，节用阿拉伯多级点号。
- **图**：按章编号 `图 2-1`、`图 2-2`…（章号-序号），图题在图下方居中。
- **表**：按章编号 `表 3-1`…，表题在表上方居中；表用三线表（见 `docx-assembly.md` §3）。
- **公式**：按章编号、右对齐括号编号 `公式(2-3)` 或 `(2-3)`；正文引用"由公式(2-3)"。
- 正文每个图/表/公式都要被引用一次；**编号按出现顺序连续，引用与存在一一对应**（断号/孤引=硬伤）。
- 与期刊稿的关键差异：期刊全文连续（图1、表1），**学位论文按章**（图2-1、表3-1）。

## 5 目录（三级，自动生成）

三级标题（章 / 1.1 / 1.1.1）+ 页码右对齐 + 制表位前导点。**用 Word 目录域自动生成**（标题套 Heading 1/2/3
样式后插 TOC 域），不手敲页码。综述、致谢、参考文献、附录作为一级条目入目录。前置罗马页码段与正文阿拉伯段
分别计数。

## 6 摘要 / 缩略语 / 参考文献排版

- **中英摘要**：各自另起页；"摘 要 / Abstract"作居中标题；结构式段落小标题（研究背景/目的/…）加粗；关键词
  另起一行、`【关键词】`加粗、分号隔。
- **缩略语表**：两列表格（缩写｜全称），按字母序，五号。
- **参考文献**：GB/T 7714，悬挂缩进（`first_line_indent=-21pt`、`left_indent=21pt`），按正文出现顺序编号，
  与正文上标引用一一对应。

## 7 python-docx 拼装学位论文：差异点（补 docx-assembly.md）

`docx-assembly.md` 给通用 md→docx 实现；学位论文额外需要：
1. **多文件按固定顺序合并**：拼装脚本显式列出 sections/ 部件顺序（封面→声明→中文摘要→英文摘要→缩略语→目录
   占位→各章→参考文献→综述→致谢→在读成果→附录），不依赖文件名字典序。
2. **分页符**：每个一级部件之间插 `add_page_break()`（封面、声明、摘要、目录、每章、参考文献、综述、致谢、附录）。
3. **双页码段**：用 section 分节，前置部分页码格式罗马、正文段 `start=1` 阿拉伯重新计数（`w:pgNumType`）。
4. **三级标题套 Heading 1/2/3 样式**（手动构造样式，字号字体按 §3），供 TOC 域识别。
5. **TOC 域**：在目录页插入 `TOC \o "1-3"` 域，提示用户在 Word 内按 F9 更新页码。
6. 三线表、OMML 公式、真上下标、图片+图注同 `docx-assembly.md`。
7. 拼装脚本是一次性文档脚本 → 跑完归 `09_backup/<日期>_scripts_oneoff/`，不留无编号脚本在 `02_code/`。

## 8 加亮占位（人补处）的实现

`chinese-thesis.md` §8 规定两类需人补标记：`[待补充：xxx]`、`[NEED CONFIRMATION：xxx]`。拼装时让它们在
docx 里**醒目可查**：
```python
from docx.enum.text import WD_COLOR_INDEX
run = p.add_run(placeholder_text)
run.font.highlight_color = WD_COLOR_INDEX.YELLOW   # 黄色高亮，作者一眼定位
```
拼装结束扫描全文统计两类标记数量，打印清单（部件 + 行 + 标记内容），随交付汇报给用户，作为"待人工补全项"。

## 9 拼装前后自检

拼装前：sections 部件齐全；各部件自检全过；摘要数字 = 结果 = `0_result_summaries.md`；`[待补充]`/`[NEED
CONFIRMATION]` 已就位（该占位的占位、不杜撰）。
拼装后：.docx 生成成功；分页正确（各部件另起页）；前置罗马 / 正文阿拉伯页码分段正确；三级标题样式正确、TOC
域可更新；字体字号符合 §3 / 学校规范；图表按章编号、公式非乱码、三线表线条正确；黄色高亮占位可见且已计数。
