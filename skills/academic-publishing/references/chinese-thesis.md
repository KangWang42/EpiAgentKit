# 中文学位论文写作 reference（硕士 / 博士，GB/T 7713.1）

> 与 `chinese-paper.md`（期刊论著，≤5000 字）并列、互斥：**学位论文是长文**（硕士常 5–8 万字 / 80–120
> 页，博士更长），结构、字数、部件、排版都与期刊论著不同。本文件管"结构 + 各部件该写什么 + 字数 + 自检"；

> 去 AI 味用 `chinese-anti-ai.md`；硬伤清单用 `review-killers.md`。

## 1 最高法：学校格式规范优先

期刊论文以"目标期刊 Guide for Authors"为最高法；**学位论文以"本校研究生院最新《学位论文格式规范/写作
指南》"为最高法**。本文件给的是 GB/T 7713.1 + 常见高校通行基线（字数区间、部件、编号），**一旦用户给出
本校规范或本人/本专业的具体要求（字数、字体、页边距、参考文献格式、章节编号），其覆盖本文件一切默认值**。
学校规范未知 → 先问用户要，或先按基线写并在交付时标注"排版默认值待以学校规范核对"。

可参考同实验室 / 同导师 / 同专业的已通过学位论文作为**结构与排版范本**（学其框架，不抄其内容）。

## 2 整体结构与 sections/ 分文件落位

学位论文长，**逐部件写、各存独立 md**（与项目 `07_paper/sections/` 一致），拼装时按序合并。推荐文件方案
（按论文行文顺序编号，可随研究类型增删）：

```
07_paper/sections/
  01_metadata.md      封面信息 + 题名(中/英) + 作者/导师/专业/答辩委员会 + 关键词
  02_abstract.md      中文摘要 + 英文摘要(Abstract) + 中英关键词
  03_introduction.md  第 1 章 前言（背景 / 现状 / 问题 / 目的意义）
  04_methods.md       第 2 章 方法（对象 / 随访 / 样本量 / 预处理 / 变量 / 模型 / 技术路线）
  05_results.md       第 3 章 结果
  06_discussion.md    第 4 章 讨论（+ 结论，或结论单列 07）
  07_references.md     参考文献
  08_appendix.md      附录（报告清单 / 量表 / 代码 / 补充表图 / 在读成果）
  09_review.md        独立综述章
  10_acknowledgements.md  致谢（如单列）
```

> 缩略语表、致谢、在读成果、结论可单列文件，也可并入相邻部件——**与本项目既有 sections/ 文件划分保持一致**，
> 不擅自改名/拆并。拼装顺序由拼装脚本固定（见 `thesis-formatting.md`），不依赖文件名字典序。

**全文部件总览（GB/T 7713.1 顺序）**：封面 → 学位论文原创性声明 → 使用授权声明 → 中文摘要 → 英文摘要 →
（缩略语表）→ 目录 → 正文各章 → 结论 → 参考文献 → 综述 → 致谢 → 在读期间成果 → 附录。

## 3 各部件页数 / 字数目标（硕士基线 80–120 页；学校规范优先）

| 部件 | 页数基线 | 字数基线 | 备注 |
|---|---|---|---|
| 中文摘要 | 1–2 | 800–1500 | 结构式分段 + 关键词 5–8 个 |
| 英文摘要 | 1–2 | 与中文对应 | 内容与中文摘要逐段对应 |
| 缩略语表 | 1 | — | 全文缩写 ≥5 个时设 |
| 第 1 章 前言 | 5–8 | 4000–7000 | 4–5 节 |
| 第 2 章 方法 | 15–25 | 8000–15000 | 最长章；含公式编号 / 技术路线图 |
| 第 3 章 结果 | 20–30 | 8000–15000 | 主体表图集中于此 |
| 第 4 章 讨论 | 8–12 | 6000–10000 | 多节结构 + 研究特色 + 不足 |
| 结论 | 1–2 | 600–1200 | 逐条定性，无统计量 |
| 参考文献 | 4–8 | 30–80 条 | GB/T 7714 |
| 综述（独立章） | 8–12 | ~6000 | 30+ 引用，主题区别于正文 |
| 致谢 | 1 | 300–600 | — |
| 在读成果 | 1 | — | 论文 / 专利 / 课题 |
| 附录 | 视情况 | — | 报告清单 / 量表 / 关键代码 / 补充表图 |

字数是区间不是靶心；**学校有硬性下限（如正文≥3 万字）时以学校为准**。任一部件低于下限 → 内容不足，回看
`section-content-playbook.md` 补"该写而没写"的内容，不靠注水。

## 4 前置部件逐一写法

### 4.1 封面 / 题名（01_metadata.md）
中英文题名、专业、研究生、导师、答辩委员会、年月、关键词。**作者真实信息（姓名/单位/导师/批号/答辩委员）
一律 `[待补充：xxx]` 加亮占位**，不杜撰（见 §8 加亮约定）。题名是名词短语、含研究类型词（如"…的预测研究"
"…的横断面研究"），不超 25 字，不用缩写。

### 4.2 原创性声明 / 使用授权声明
学校固定模板，**逐字采用本校官方版本**；本文件不提供模板原文，交付时以 `[采用本校官方声明模板]` 占位提示，
或由用户粘贴。签名、日期留空 `[待签名]`。

### 4.3 中文摘要（02_abstract.md 上半）
**结构式分段、用段落小标题**（学位论文惯例，区别于期刊的【目的】【方法】方括号标签）：
```
摘 要
研究背景  （1–2 句交代疾病/问题重要性与现存空白）
研究目的  （本研究做什么、解决什么）
研究对象与方法  （设计 + 对象来源 + 时间地点 + 样本量 + 关键变量/工具 + 主分析方法清单 + 软件）
研究结果  （分条 1.2.3.…，每条给关键数字：估计值±/率/AUC/CI/P，与结果章一致）
研究结论  （分条，文字定性，落到意义；不含 β/P/CI 细节）
【关键词】词1；词2；…（5–8 个，分号隔）
```
摘要每个数字必须等于结果章与 `0_result_summaries.md`，不得出现源里没有的数。

### 4.4 英文摘要 Abstract（02_abstract.md 下半）
与中文摘要逐段对应（Background / Objective / Methods / Results / Conclusions + Keywords）。时态、over-claim
黑名单走 `english-phrasebank.md`。专业术语首次给全称。中英不一致 = 硬伤。

### 4.5 缩略语表
两列（缩写｜中英文全称），按字母序。全文每个缩写首次出现给全称，此后用缩写并与表一致。

### 4.6 目录
三级标题自动生成（见 `thesis-formatting.md`），不手敲页码。

## 5 正文各章写法

各章"具体写什么、详略、句式、最易写错处"复用 `section-content-playbook.md`（统计分析写成编号清单、量表五要素、
讨论影响因素逐个成节、结论逐条无统计量等）。学位论文相对期刊的**加写要求**：

### 5.1 第 1 章 前言（4–5 节）
背景 → 国内外研究现状（分国外/国内）→ 存在问题 / 研究空白 → 研究目的与意义（→ 可附研究内容/技术路线概述）。
比期刊引言**更展开**：现状部分是真综述（有文献对照、指出空白），不是三句话漏斗。

### 5.2 第 2 章 方法（学位论文的重头，最长章）
逐节：研究对象（纳入/排除/样本量计算，**样本量给公式与 α/β/效应量**）→ 随访 / 数据采集 → 变量与测量 →
预处理 → 统计 / 建模方法 → 模型评价 → 质量控制 / 伦理 → **技术路线图**。
- **每个关键公式编号**（如"公式(2-3)"），写成 LaTeX `$$...$$`（拼装转 OMML）。
- 关键流程 / 模型结构 / 抽样配对**给示意图或技术路线图**。
- 写"做了什么"够复现即可，不写方法学定义、不写动机辩解（见 `section-content-playbook.md` §4）。

### 5.3 第 3 章 结果（主体表图集中）
按研究问题分节，先点表图后述关键数字，纯客观不解释。学位论文**表图比期刊多、可多角度展示**（敏感性、亚组、
分层），但每张图表都要被正文引用、按出现顺序连续编号（图2-1、表3-2…）。差结果如实报告，不删。

### 5.4 第 4 章 讨论（多节结构）
比期刊讨论**更分层**，建议：主要发现 → 与既往研究对照 → 机制 / 解释（逐点）→ 临床或应用意义 → **研究特色/
创新**（可分子节）→ **研究不足 + 未来方向**（每条=不足点+影响+应对）。审慎措辞，观察性不下因果。

### 5.5 结论（独立，逐条定性）
全文收口，逐条概括"做了什么、得到什么、有何意义"，**不含 β/P/CI**，不复制结果细节，不据探索性结果提具体
分层方案。一段或分条均可，火候在不单薄也不臃肿。

## 6 独立综述章

学位论文的综述**是独立章、主题须与正文研究区别开**（综述 = 领域方法/背景的系统梳理，不是讨论的扩写）。
5–7 节漏斗结构：领域背景 → 发展脉络 → 关键方法/技术分支 → 现有代表性工作对比 → 应用场景 → 挑战与未来。
~6000 字、30+ 引用。引用为主、述评结合，避免变成教科书摘抄。

  附录前缀（表 A-1、图 B-2）。

## 8 长文生成工作流（门控 + 分文件 + 加亮人补）

1. **逐部件门控**（同 SKILL.md §2.3）：一次只写一个部件 → 自检 → 字数核对 → 标记完成 → 下一个。长文**禁止
   一次吐全文**；先 Methods/Results，再 Introduction/Discussion，再 Abstract/Title，最后综述/致谢/附录。
2. **各部件写入独立 md**，不动其他部件文件；改已认可部件用最小修改。
3. **目标=可直接用的完整稿**：每部件按本文件 §3 字数与 §5 要求写满写实，不留"此处略""TODO 待写"空壳。
4. **需人补的一律加亮占位、集中可查**，两类标记：
   - `[待补充：xxx]` —— 作者无法由数据/代码推出、必须本人提供的真实信息（姓名、单位、导师、伦理批号、
     基金号、答辩委员、签名、致谢对象、在读成果）。
   - `[NEED CONFIRMATION：xxx]` —— `0_result_summaries.md` 里查不到的数字/结论，不许瞎填。
   拼装时这两类标记在 docx 里**黄色高亮**（见 `thesis-formatting.md`），并在交付汇报中给出两类标记的清单与计数。
5. **数字唯一来源 = `0_result_summaries.md`**；源不自洽（新旧值混杂）先让用户统一再写，不在混杂源上动笔。
6. 结果变 → 回写 `0_result_summaries.md`；方法变 → `DECISIONS.md`；操作完 → `SESSION_LOG.md`。

## 9 与期刊论著的差异对照

| 维度 | 期刊论著（chinese-paper.md） | 学位论文（本文件） |
|---|---|---|
| 篇幅 | ≤5000 字 / 几页 | 5–8 万字 / 80–120 页 |
| 摘要 | 【目的】【方法】方括号标签，300–500 字 | 段落小标题（研究背景/目的/…），800–1500 字 + 英文摘要 |
| 章节 | 前言-方法-结果-讨论-结论，层级≤2 | 第1–N章，层级到 3 级；含独立综述章 |
| 综述 | 融在引言 | **独立章**，主题区别于正文 |
| 公式/图表编号 | 全文连续（图1、表1） | **按章**（图2-1、表3-2、公式2-3） |
| 前后置 | 无声明/致谢/在读成果 | 封面+双声明+缩略语+致谢+在读成果+附录 |
| 最高法 | 期刊 Guide for Authors | 本校研究生院学位论文格式规范 |
| 排版 | 投稿系统多自带 | 国标级，作者自排（见 thesis-formatting.md） |
