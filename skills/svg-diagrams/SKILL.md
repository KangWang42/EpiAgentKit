# SVG 图解

以原生 SVG 构建简洁、优雅、清晰、可编辑的非统计图形。默认不调用 R 绘图包生成流程框，也不把 HTML、Mermaid 或 Graphviz 默认主题截图当成品。

## 工作流

1. 锁定信息：逐项列出节点、文字、数字、关系、顺序和分支；不增添副标题、英文眉题、图标或解释句。
2. 选择拓扑：线性流程、分支流程、包含关系、层级树、时间线、矩阵、机制图或架构图。非平凡图先读 `references/design-system.md` 对应章节。
3. 计算几何：先写画布、边距、行列、节点宽高、锚点和连接线坐标，再写 SVG。禁止凭目测逐个拖位置。
4. 生成 SVG：使用 `<rect>`、`<path>`、`<line>`、`<text>`、`<tspan>` 等原生元素；保留可编辑文字；避免 `foreignObject`、外部 CSS、滤镜堆叠和依赖网络的字体。
5. 导出预览：SVG 是当前源文件，同时生成同名 PNG 供 Word、缩略图和目视检查。期刊要求 PDF/EMF 时从 SVG 派生，不反向从 PNG 描摹。
6. 验证：运行 `scripts/validate_svg.py`，再实际渲染并查看整图；发现一类对齐或字号问题后全文检查同类元素。

## 画布与载体

- PPT 全页图：默认 `viewBox="0 0 1600 900"`，16:9；嵌入半页或栏位时按实际占位比例生成，不把方图硬塞进宽框。
- 论文图：按最终单栏、1.5 栏或双栏宽度设计；常用双栏宽 180 mm，高度随信息密度确定。图题与解释留给正文，不嵌入图内。
- 报告图：按 Word 版心宽度设计，通常 140–165 mm；保留同名 PNG 作为 python-docx 的嵌入回退。
- 所有载体：内容决定高度，避免为了填满固定比例制造大面积空白。

## 默认视觉规则

- 只保留一个视觉主张。无用户要求时，不添加英文眉题、副标题、装饰圆点、无语义图标或背景大色块。
- 载体已有标题时，SVG 内不重复放标题；PPT 页标题、论文图注或报告小节标题与 SVG 内标题二选一。
- 框就是框：默认淡色或白色背景、1–1.5 px 细边框、适度圆角；不加顶部深色色条，不用厚重阴影。
- 强调靠字重、留白和有限色彩，不靠彩虹色。全图使用 1 个主色、1–2 个辅助色和中性色。
- 同层节点等宽等高；行列共用锚点；相邻间距一致；分支使用正交或短曲线连接，避免长斜虚线跨越大片空白。
- 序号标记与标题第一行的视觉中心在同一水平线上。序号、标题和副文字不能各自漂浮在不同基线。
- 分支标题、组标题和说明文字不得小于同图正文；小字不能承担结构层级。
- 包含关系图按共同中心或共同基线嵌套，各层内边距递进一致；层标题与样本量在同一基线，底部说明居中且与边框保持安全距离。
- 中文字体统一使用思源黑体、苹方或微软雅黑之一；英文使用 Arial/Helvetica。SVG 中设置字体回退链，不依赖单一机器字体。

## 与论文、报告和 PPT 的衔接

- 统计图、森林图、生存曲线、热图等仍由 `publication-figures` 生成；流程、结构、机制、路线和包含关系图由本技能生成。
- 论文和报告保留 SVG 源文件与同名 PNG；Word 不直接支持时嵌入 PNG，但不得丢失 SVG 源。
- PPT 优先直接嵌入 SVG；若当前生成库不能嵌入，才使用同名 PNG 回退，并在源目录保留 SVG。
- 混合图先分别生成统计面板和 SVG 图解面板，再按共同字体、配色和边距拼装。

## 强制自检

- [ ] 文字、数字和关系与来源逐项一致，无擅自新增内容
- [ ] 画布比例匹配最终载体，无明显空边或压缩变形
- [ ] 同行、同列、同层、嵌套框和连接锚点严格对齐
- [ ] 序号与标题第一行垂直居中对齐
- [ ] 字号层级清楚，分支标题不小于正文
- [ ] 无英文眉题、副标题、深色顶条、装饰圆点和无语义图标，除非用户明确要求
- [ ] 连接线不穿字、不穿框、不交叉，箭头方向明确
- [ ] SVG XML 有效，文字未裁切，PNG 预览与 SVG 一致
- [ ] 当前版使用稳定文件名，旧版已整组归档

验证示例：

```bash
python skills/svg-diagrams/scripts/validate_svg.py output.svg \
  --expected-ratio 1.777778 --require-text "主流程" --forbid-text "RESEARCH WORKFLOW"
```
