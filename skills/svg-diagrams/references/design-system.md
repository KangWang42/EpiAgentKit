## 1. 视觉原则

- 层级先于装饰：标题、组标题、节点标题和说明文字用字号、字重和位置形成层级。
- 对齐先于美化：任何元素都要能说明它与哪条水平线、垂直线、中心线或基线对齐。
- 邻近表达归属：属于同组的元素距离更近，组间距离更大。
- 重复建立一致性：同层节点共享宽高、圆角、内边距、标题位置和行距。
- 对比保持克制：强调色只标识关键节点或类别，不让每个框竞争注意力。
- 留白服务阅读：边距和间距要规律；既不塞满，也不留下与结构无关的大面积空白。

## 2. 对齐与几何

先建立设计令牌，再计算坐标：

| 元素 | 默认范围 |
| --- | --- |
| 外边距 | 画布短边的 5%–8% |
| 同层节点间距 | 节点宽度的 18%–30% |
| 框内水平边距 | 文字高度的 1.2–1.8 倍 |
| 框内垂直边距 | 文字高度的 0.9–1.4 倍 |
| 圆角 | 框高的 8%–14% |
| 边框 | 1–1.5 px；打印图按最终尺寸复核 |
| 主连接线 | 2–3 px |
| 次连接线 | 1.5–2.5 px |

### 2.1 水平流程

- 同层卡片共用 `y`、`height` 和垂直中心。
- 箭头从前框右侧中心连到后框左侧中心，线段长度一致。
- 步骤序号与标题第一行共用视觉中心：圆心 `cy` 对齐标题首行的中线，不把序号悬在左上角、标题另放在下方。
- 一行放不下时优先减少节点宽度差异或改两行流程，不缩到难读。

### 2.2 垂直与分支流程

- 主干保持一条中心线；分支点落在主干中心。
- 分支线优先正交：先垂直、再水平、再垂直进入节点。
- 分支标签直接占据结构位置，不额外添加圆点。标签字号至少等于节点副文字。
- 两个并列分支的卡片宽高、顶部和中心线一致。

### 2.3 包含关系图

- 外层、中层、内层矩形共享中心，或共享一组明确的左右边距；禁止每层凭目测偏移。
- 各层内边距按固定级差递进，例如 72、60、48 px，而不是忽大忽小。
- 层标题与样本量放在同一文本基线；样本量紧随标题但保持固定间距。
- 内层说明按文本块整体垂直居中，行距一致；底部来源说明在外框内居中，并与下边界保留至少一行字高。
- 使用同一色系的浅色层级或一主两辅，不用厚边框加高饱和底色制造“套盒”。

### 2.4 矩阵、结构与架构图

- 先确定列轨和行轨，所有模块贴轨排列。
- 父级容器只表达范围，不与子节点争夺颜色和边框权重。
- 输入、处理、输出等语义用位置和标签表达，颜色只作辅助。
- 跨层连接尽量沿容器边缘走线；禁止连线穿过文本和节点内部。

## 3. 字体与文字

- 中文字体链：`"Source Han Sans SC", "Microsoft YaHei", "PingFang SC", sans-serif`。
- 英文字体链：`Arial, Helvetica, sans-serif`。
- PPT 全页 1600×900 参考：页标题 34–40 px，节点标题 22–26 px，正文 16–20 px，组标题 18–22 px，步骤序号 13–16 px。
- 论文和报告按最终毫米宽度换算，最小正文不低于最终 7–8 pt。
- 无明确要求时不自动添加英文标题、英文眉题、副标题、口号或解释句。
- PPT 页、论文正文或报告章节已经提供标题时，SVG 内不再重复标题；独立查看的 SVG 才按需保留一个主标题。
- 文本超过两行时增大框高或拆分节点，不靠压缩字距、极小字号或溢出解决。
- SVG 保留 `<text>` 与 `<tspan>`，不把全部文字转路径；交付前确认目标机器有字体回退。

## 4. 色彩与框

默认用浅背景、细边框和深色正文：

- 画布：`#FAFBFC` 或白色。
- 正文：`#1D303A`；次文字：`#576A75`。
- 中性框：填充 `#F3F5F4`，边框 `#B8C2BE`。
- 蓝色框：填充 `#F1F7FA`，边框 `#9CBED1`。
- 金色框：填充 `#FFF9ED`，边框 `#D9BD7C`。
- 绿色框：填充 `#F1F8F5`，边框 `#9CCABD`。

默认禁止：

- 框顶部深色色条或厚重色块
- 多层投影、发光、玻璃效果和 3D
- 无语义装饰圆点、背景大圆、图标堆叠
- 五个节点使用五种同等饱和的颜色
- 红绿作为连续层级或唯一可区分方式

## 5. 连线

- 主流程使用一致的箭头大小、线宽和颜色。
- 分支连接使用次一级中性色，不用长斜虚线跨越空白。
- 箭头终点与框边保留 6–12 px 视觉间隙，避免箭头压框。
- 圆角折线路径的转弯半径一致。
- 双向关系、反馈环和可选路径必须有明确图例或文字标签。

## 6. 图型配方

### 6.1 线性流程

一行 3–6 个等宽卡片。序号与标题同行，副文字另起一行。超过 6 步改两行蛇形流程或分阶段。

### 6.2 主流程加扩展分支

主流程占上半部；从关键节点向下接一个字号充分的组标题，再正交分至并列扩展卡片。不要用两条长斜虚线直接跨到画布底部。

### 6.3 包含关系

使用同心或同基线嵌套框表达集合关系。外层标题、样本量和来源说明分别占据顶部、中心与底部安全区，内层框居中且内边距递进。

### 6.4 时间线

时间轴只保留一条主线；事件标签交替上下或分行，时间点与标签共用垂直锚点。长说明改为旁注卡片，不沿轴堆字。

### 6.5 概念框架与机制图

核心概念居中，输入在左、机制在中、结果在右；相同语义层使用相同形状。因果箭头、关联线和调节路径用不同线型并给图例。

### 6.6 矩阵与架构图

行列标题占固定轨道；模块对齐网格；容器层使用淡边框。跨区连接沿通道走线，避免穿过模块。

## 7. 载体适配

- PPT：全页图 16:9；右栏图按栏位比例生成。直接嵌 SVG，演示软件不支持时用同名 PNG。
- 论文：图题在正文；SVG 本体不放标题。按 88–90、120–140 或 180 mm 宽度设计，并提供期刊要求的 PDF/PNG 派生文件。
- 报告：SVG 源与 PNG 同名共存；docx 使用 PNG 回退，Markdown 链接优先指向 SVG 或 PNG，保持一致。

## 8. 目视检查

将 SVG 渲染为 PNG 后逐项检查：

1. 缩略图下能否一眼看出主次和方向。
2. 所有同层框是否共线、等高、等距。
3. 序号是否与标题第一行垂直居中。
4. 包含关系各层是否真正居中，内边距是否规律。
5. 字号是否有明显跳级，组标题是否被缩小成注脚。
6. 连线是否穿字、穿框、交叉或出现无解释的点。
7. 是否存在无用户要求的英文、副标题、图标和装饰背景。
8. 最终载体中是否被拉伸、裁切或留下明显空边。
