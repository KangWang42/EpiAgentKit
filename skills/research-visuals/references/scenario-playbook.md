本参考用于 README、项目主页、技术文档、教程、skill 示例、成果展示、真实截图和已有模板任务。先选择证据与实现路线，再谈画风。模板是可调整的边界，不是固定版式。

## 1. 六项场景判定

为每个图位记录：

1. **载体**：README、技术文档、网页、PPT、论文、报告、标书或其它容器。
2. **信息功能**：解释关系、证明功能、展示成果、指导操作、建立情境、形成识别或提示状态。
3. **信息角色**：正文内容图、真实截图、氛围插图、统计图、原始证据或无需图片。
4. **证据属性**：解释示意、精确结构、真实界面、真实成果、统计数据或原始证据。
5. **模板状态**：无模板、可调整模板或必须保持的品牌系统。
6. **最终尺寸**：实际宽高、窄屏裁切、投影或打印尺寸和最小可读字号。

图位进入生成前必须回答“移除图片会损失什么理解”。没有独立贡献时删图；不能因为 README、PPT 或网页有空位就机械配图。

## 2. 文字策略

### 2.1 正文内容图

正文内容图包括工作流、架构图、技能示例、方法说明、比较图、教程步骤、机制、路线和成果路径。图按内容需要直接包含理解结构所需的短标签、数字、箭头、状态或步骤；读者不依赖相邻段落也能识别对象和关系。

文字选择先于排版，但晚于内容核验。先读取图位对应的来源材料、实际产物或规范方法，锁定对象名称、关系语义、数字和结论；来源不足且外部证据会改变图意时先走 `evidence-research`，存在多个合理术语或口径时向用户确认。不得用省略必要文字掩盖内容不确定，也不得用一条看似合理的解释补齐未核验关系。

来源锁定后做逐条删除测试。只有删除后会使对象身份、关系类型、分支条件、关键状态或核心结论产生歧义的文字才进入图内。图内文字分四层：

1. **身份标签**：对象、阶段、变量、方法或产物的规范名称。没有图标也必须能认出节点时保留。
2. **关系与条件**：仅当箭头不只是普通顺序，或存在分支条件、反馈、调整、中介、验证等特殊关系时标注。
3. **关键限定**：同类节点可能混淆、证据状态必须区分，或输出无法从标题推知时，为少数关键节点增加一条限定语。
4. **图外解释**：定义、背景、理由、完整方法、限制、来源和操作说明优先写在正文、图注或复杂图长描述中，不塞进节点。

每段候选文字依次回答：

- 删除后是否会误认对象或关系？
- 是否已被载体标题、位置、形状、图标或相邻标签清楚表达？
- 是否来自来源材料或规范术语，且不存在更短的等义表达？
- 是否能在最终显示尺寸下直接读出？
- 是否应移到图注或长描述，而不是留在图内？

只有第一项为“会”、第二项为“否”，并通过后三项的文字才进入 `Text (verbatim)`。不要给每个节点机械添加副标题；限定语只用于确有歧义或证据优先级的节点。

以下字符数是 EpiAgentKit 的本地起点，不是期刊或无障碍标准，最终尺寸和规范术语优先：

| 文字层级 | 中文启发式 | 使用条件 |
| --- | --- | --- |
| 图内总标题 | 6–18 字 | 图片需独立传播或载体没有等价标题；否则省略 |
| 阶段 / 分区标题 | 2–8 字 | 只命名信息区，不写结论句 |
| 节点主标签 | 4–12 字，必要时放宽至 14 字、最多两行 | 保留术语核心；不能为凑字数自造缩略语 |
| 箭头关系词 | 2–6 字 | 关系不是显然的时间或数据流时才加 |
| 关键限定语 | 8–22 字、每个关键节点最多一条 | 解决真实歧义，不给所有节点铺副标题 |
| 解释性短句 | 12–30 字、全图通常 0–2 条 | 只有移出图外会导致误读时保留；其余进图注 |

一个 5–7 阶段的常规流程通常只需要阶段或节点主标签、必要的分支条件，以及 1–3 条真正消歧的限定语；这只是密度起点，不是配额。标签使用来源或规范术语，不以“无伪文字”为由删除真实标签，也不为视觉对齐把所有节点补成相同字数。

### 2.2 真实截图

界面、终端、代码、文档、PPT、表格、统计图和交付包预览中的文字来自真实运行或渲染结果，不由 imagegen 生成。截图可证明“确实能做什么”和“最终产物是什么”，不能用似真 UI 替代。

### 2.3 氛围插图

Hero、封面、登录侧栏、章节氛围图和纯情境配图默认不烧录标题、正文、按钮或口号。它们为载体原生文字保留负空间，只承担识别、情境或情绪功能。

## 3. 场景路由矩阵

| 场景 | 默认实现 | 文字策略 | 关键验收 |
| --- | --- | --- | --- |
| README / 项目主页 hero | imagegen 解释插图或编辑视觉 | 默认无字，标题与按钮留在页面 | 主体与项目领域一致，裁切安全，缩略图可辨 |
| README 工作流 / 架构 | imagegen 精确结构图 | 节点、关系和必要标题直接入图 | 文字、节点、边和方向 100% 一致 |
| Skill 功能示例 | 概念路径用 imagegen；真实成果用截图 | 内容图直接带标签，截图保留真实文字 | 一眼看出输入、处理和产物，不做无字装饰 |
| 安装或操作教程 | 真实终端、IDE、网页或应用截图 | 保留必要命令与控件文字 | 步骤可复现，无敏感信息和无关窗口 |
| 代码与命令示例 | 优先 Markdown 代码块；需要环境证明时截图 | 使用真实代码或终端输出 | 不用 imagegen 伪造代码和命令 |
| 文档 / PPT / 表格预览 | 实际生成并渲染后截图 | 保留真实标题、表格和版式 | 与交付文件一致，不伪造内容 |
| 统计分析示例 | `publication-figures` 实跑真实或公开示例数据 | 坐标、单位、图例和来源完整 | 不用 imagegen 生成似真统计结果 |
| 流程、机制、路线、图形摘要 | imagegen 精确结构图 | 必要的短标签、数字或箭头直接入图 | 结构和证据边界准确，最终尺寸可读 |
| 概念教育图 | imagegen 科学教育插图 | 只有理解层级所需标签入图 | 无虚构解剖、机制或数据 |
| 功能卡片 / 空状态 | imagegen 定制插图或现有图标系统 | 页面原生文字；只有内容本身需要时入图 | 小尺寸轮廓清楚，不替代通用控件图标 |
| 前后对照 / 重绘 | 携 Image 1 定向编辑 | 原图文字与关系列入 LOCKED | 高优先级内容不降质，变化可观察 |
| 真实原始科研图像 | 保留原件并做允许的确定性处理 | 保留原始标记和来源 | 不生成式重绘或补全 |

同一 README 或文档可以混用多条路线，但相邻资产共享字体倾向、主色来源、边缘语言和留白节奏。不要为了“统一”把真实截图改造成插画，也不要把所有示例都做成同构卡片。

## 4. 模板适配合同

先把模板拆成两组：

### 必须保留

- 目标比例、页面或图位边界。
- 已确认的品牌标识、字体、主色和无障碍要求。
- 导航、标题、按钮、页码、图注等由载体原生管理的组件。
- 用户明确锁定的网格、留白方向或图片安全区。

### 允许调整

- 区域数量、列数、主次面积和阅读方向。
- 图片与文字的占比、节点数量、标签长短和图标预算。
- 颜色在不同语义角色中的分配、材质、线宽和细节密度。
- 截图裁切、示例顺序和桌面/移动端派生构图。

按内容拓扑改模板：

- **顺序流程**：使用单向主链，阶段等距，起点与产出闭合；不拆成无箭头卡片墙。
- **并列比较**：比较维度一致、视觉权重对称；不强设英雄面板。
- **层级或包含**：使用嵌套、树或共同基线；不把层级压成线性箭头。
- **多源汇聚**：来源整齐分叉后汇入唯一整合节点；不产生交叉线。
- **成果展示**：真实截图按共同基线和比例排列，突出最能证明能力的 2–4 个产物；不把所有文件都缩成不可读缩略图。
- **情境与品牌**：保持一个焦点和可用负空间；不把全部功能塞进 hero。

模板导致以下任一结果时必须重排：文字小于最终可读尺寸、关系被隐藏、内容被迫重复、主次与证据强度相反、关键对象贴近裁切边缘、图文失衡或为了填满而加入无贡献元素。

## 5. 内容拓扑与美学实现

美观来自关系被看清，不来自装饰数量：

| 内容关系 | 美学重点 | 避免 |
| --- | --- | --- |
| 单向路径 | 稳定节奏、清楚箭头、阶段间呼吸 | 断裂连线、每节点不同颜色 |
| 比较 | 共同网格、对称尺度、相同维度 | 一边复杂一边空、无依据强调 |
| 层级 | 尺度递进、边界清楚、留白递增 | 所有层级同字号同权重 |
| 总览加局部 | 母图定位、局部只展开一个模块 | 多处同时放大、重复全部内容 |
| 证据与文献 | 纸张、索引、矩阵和批注的克制语言 | 伪论文段落、复古道具堆砌 |
| 数字系统 | 网格、矩阵、真实微型图元和清楚连接 | 全息屏、代码雨、通用 AI 脑 |
| 现场与公共卫生 | 自然光、可信动作、普通工作环境 | 商业摆拍、完美微笑、戏剧光效 |
| 成果集合 | 真实缩略图、共同基线、有限重点 | 等权文件墙、不可读小图 |

每套资产建立最小视觉系统：一个背景倾向、一套主色来源、一个强调色逻辑、一种主要线条或摄影语言。相邻图保持家族一致性，但允许根据内容关系改变布局、密度和对象尺度。

## 6. 真实截图合同

截图前先运行或打开真实产物，并锁定：

- **证明目标**：截图具体证明哪个功能、步骤或产物。
- **真实状态**：文件、命令、界面、结果和版本来自当前可复核环境。
- **可见范围**：只保留理解目标所需的窗口、面板和上下文。
- **隐私边界**：隐藏姓名、路径、账号、令牌、通知、历史记录和非公开数据。
- **视觉整理**：统一缩放、窗口尺寸、主题和光标状态；关闭无关弹窗。
- **标注规则**：只用必要的编号、框线或箭头指出真实控件，不覆盖原内容，不把标注写成新的界面状态。

实际成果预览需要多个步骤时，优先 2–4 张连续截图或一张带原生步骤编号的内容图。单张截图无法说明因果顺序时，不用装饰箭头臆造流程。

## 7. 系列资产与验收

交付前逐项检查：

- [ ] 每个图位已选定正文内容图、真实截图、氛围插图、统计图、原始证据或无需图片。
- [ ] 正文内容图含理解关系所需的准确短标签；hero、封面和氛围图才默认无字。
- [ ] 真实界面、代码、终端、文档和成果来自实际渲染，不是生成式伪造。
- [ ] 模板只保留必要约束，结构、层级、密度、留白和图文比例已按内容优化。
- [ ] 每张图只承担一个主要功能，相邻图无重复内容或机械同构。
- [ ] 图内文字、节点、边、数字和身份与来源逐项一致。
- [ ] 最终显示尺寸下文字、箭头、主体和截图内容仍可读。
- [ ] 桌面与窄屏裁切保留关键内容；图片具有适当替代文本和合理文件体积。
- [ ] 无伪 UI、伪代码、伪论文、假统计结果、无关科研符号和模板化装饰。
- [ ] 图内文字通过逐条删除测试；没有机械副标题，长解释已移到正文、图注或复杂图长描述。

## 8. 依据与本地启发式边界

- [PLOS Computational Biology: Ten Simple Rules for Better Figures](https://journals.plos.org/ploscompbiol/article?id=10.1371/journal.pcbi.1003833) 强调先识别受众与核心消息，并按论文阅读或限时演示等真实观看方式删减细节、调整字号与线条。
- [PLOS Computational Biology: Ten Simple Rules for Designing Graphical Abstracts](https://journals.plos.org/ploscompbiol/article?id=10.1371/journal.pcbi.1011789) 要求先锁定面向受众的核心消息，再按最终尺寸、阅读方向和箭头语义组织视觉与必要文字。
- [CDC Clear Communication Index User Guide](https://www.cdc.gov/ccindex/pdf/clear-communication-user-guide.pdf) 要求视觉与文字共同支持一个明显主消息，使用标签与图注降低误解，同时避免视觉过载。
- [W3C WAI Complex Images](https://www.w3.org/WAI/tutorials/images/complex/) 把流程图、组织图和依赖图片理解正文的插图视为复杂图，要求短描述加等价的详细文本，并建议减少不必要复杂度。

这些来源支持“核心消息优先、标签与视觉相互强化、复杂信息提供图外等价文本、按受众和观看条件控制细节”的原则。上表字符数、限定语数量和阶段密度是为中文科研视觉建立的本地操作性启发式，不得包装为来源机构的硬标准；目标期刊、机构模板、实际像素尺寸和规范术语长度优先。
